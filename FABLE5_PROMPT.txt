══════════════════ ФАЗА 2 — ОХОТА ПО ТАКСОНОМИИ ДЕФЕКТОВ ══════════════════
Ищи СИСТЕМНО, по классам (все реально встречались в этом проекте):
  1. «Введено, но не сохранено»: поле формы собирается, но (а) не отправляется,
     (б) бэкенд не читает, (в) читается но нет INSERT/UPDATE, (г) edit-путь
     создаёт дубликат или молча глотает 409 (catch(_){}), (д) затирается
     дефолтом/нулём при сохранении соседнего поля (COALESCE(0,...)-ловушка,
     SELECT не возвращает колонку => edit-диалог шлёт 0).
  2. Дрейф схемы: код/фронт требует колонку/таблицу/значение CHECK, которых нет
     на живой БД (пример: salary_type='shift' падал по констрейнту).
  3. Контракт FE↔BE: имена полей ответа/запроса не совпадают; необработанные
     статусы; null-касты (as num на nullable JSON => красный экран).
  4. Денежная математика: двойной счёт (зарплата в expenses И в вычете),
     неоплаченный долг в прибыли, перезапись цены вместо средней, потеря
     точности (toStringAsFixed(0) на цене), NaN => null в JSON.
  5. Границы времени: полночь vs бизнес-день vs открытие кассы; backdate
     операций (created_at связанных записей должен совпадать).
  6. Права: requireRole-списки (director/guest забыты?), guest-only функции,
     официант видит чужое?
  7. Машина состояний: заказ/стол/долг/смена — из каждого состояния есть выход?
     архивная запись где-то осталась «живой» (пример: П/Ф архивирован, а его
     menu_item активен)?
  8. Конкурентность: FOR UPDATE, идемпотентность, двойной client.release,
     параллельные оплаты/переносы.
  9. Офлайн: всё ли работает без интернета (шрифты PDF из сети — уже чинили,
     ищи аналоги: сетевые ресурсы, таймауты, локальный Wi-Fi путь).
 10. UX-трение по миру актёров: застрявшие нули, лишние клики, Enter не ведёт
     дальше, смешанные языки в терминах, непонятные бизнес-термины.
 11. ДАННЫЕ vs код: невозможные yield_kg, блюда с нулевой себестоимостью,
     дубликаты — НЕ «чинить» молча: отчёт владельцу с конкретикой.
 12. Тихие ломатели сборки: dup-ключ в lang.dart тихо валит flutter build web —
     поэтому web-сборка обязательна после каждого изменения фронта.
